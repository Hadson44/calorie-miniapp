# Project structure

Репозиторій залишає `index.html` у корені, щоб додаток і далі можна було просто віддавати як статичний сайт.
Кореневі `app.js` і `style.css` тепер виконують роль тонких проксі, а основний код живе в `src/`.

## Where things live
- `index.html` — HTML-entrypoint для Telegram WebApp.
- `src/scripts/app.js` — логіка інтерфейсу, локального стану та взаємодії з UI.
- `src/styles/style.css` — уся основна візуальна система.
- `assets/images` — локальні зображення.
- `assets/icons` — іконки й майбутні SVG/PNG-ресурси.
- `assets/fonts` — локальні шрифти, якщо вирішимо відмовитися від системного стеку.
- `data` — винесені масиви, конфіги або JSON-дані.
- `docs` — короткі технічні рішення й домовленості по структурі.

## Working agreement
- Нову логіку додаємо в `src/scripts/app.js`.
- Нові стилі додаємо в `src/styles/style.css`.
- Нові локальні ресурси кладемо в `assets/` замість кореня.
- Дані, які розростаються і заважають читати код, поступово переносимо в `data/`.
